How It Works

When Netflix delivers subtitles to your browser, the script intercepts the data, splits it into chunks, and sends each chunk to your chosen AI for translation. Translated subtitles appear as an overlay on the video.

Fast Start — the first chunk is sent at half size so you see results as quickly as possible. The full-size pass follows, refining those lines with more context and validation.

Seek-aware — if you skip ahead mid-translation, the script reprioritizes chunks near your new position so what you're watching gets translated first.

Validation & retry — every chunk is checked for missing lines, wrong-language output, and formatting issues. Problematic lines are automatically retried with diagnostic feedback to the AI. Remaining issues are flagged.

Speaker name normalization — as chunks are completed, the script learns how source-language character names map to their target-language forms. It picks canonical names (using cast data when available) and applies them consistently across all chunks.

Caching — translations are saved per subtitle track. Rewatching, refreshing, or coming back later loads instantly with zero API calls. Cache is keyed by provider, model, target language, and subtitle content — changing any of these starts fresh.

Settings Reference

Provider & Model — which AI translates. These providers have free tiers with no credit card required:
  • Gemini — recommended starting point. Get a key at aistudio.google.com/apikey. Not available in EU/EEA/UK/CH.
  • Groq — very fast inference, highest free daily quota. Get a key at console.groq.com
  • Mistral — strong multilingual. Phone verification required. console.mistral.ai
  • OpenRouter — many free models through one API. openrouter.ai
  • Ollama — fully local & private, no data sent anywhere. Runs on your machine. ollama.com
Target Language — the language you want subtitles in. Be specific — "Brazilian Portuguese" works better than just "Portuguese".
Chunk Size — subtitle lines sent per request (default varies by provider). Smaller = faster per request, more requests total. Larger = fewer requests but slower first results. Most users don't need to change this.
Second Model (Advanced) — run a second, stronger model after the first pass. Example: a fast small model first, then a larger model for cleanup. Two modes:
  • Cleanup mode — re-does only lines flagged as problematic by the validator
  • Full pass mode — re-does all lines in the background (higher quality, more API calls)
Character Names — fetches show metadata (title, episode info, character names from Cinemeta) and includes it in the AI prompt. Helps produce consistent character names. Recommended for anime and shows with non-Latin names.
Fast Start — sends a smaller first chunk so translations appear sooner. The full pass then re-processes those lines with validation and more context. On by default.
Per-chunk Glossary — after each chunk, extract character names and key terms, then feed them forward into subsequent chunks. Improves consistency within an episode without extra API calls upfront.
Upfront Glossary — scan the full subtitle script for names and terms before translating begins. One extra API call, but produces better consistency from the very first chunk. Best for shows with many character names.
Source Language — override auto-detection. Useful for anime (set "Japanese") or mixed-language content. Blank = auto-detect.

Advanced Settings

Click Show advanced settings in the panel to reveal additional options:

  • Chunk Size — lines per translation request
  • Second Model — cleanup or full-quality second pass
  • Subtitles toggle — show/hide the overlay independently of translation
  • Dual subtitles — display original above translation
  • Orig on flagged — show original text for uncertain lines
  • Transcript view — open scrollable full-episode transcript
  • Font Size — subtitle overlay size
  • Timing Offset — shift all subtitles earlier or later in milliseconds
  • Retry Current Chunk — retranslate the few lines currently on screen
  • Retry All — retranslate the entire episode from your current position (ignores cache)
  • Clear Cache — delete saved translations for this episode

Troubleshooting

Subtitles aren't appearing

Enable subtitles in Netflix first (speech bubble icon → select a track). The translator intercepts Netflix's subtitle data — no source subtitles means nothing to translate.

Rate limit / quota errors

Per-minute limit: the script backs off and retries automatically — no action needed. Daily quota exhausted: consider switching to Groq (14,400 req/day free) or adding it as a second provider. A banner will appear offering to swap models when a rate limit is hit.

Some lines look wrong

Press R to retry the current chunk or Shift + A to redo everything from your current position. Enabling Character Names or Upfront Glossary in Advanced often helps with name accuracy. For anime, set Source Language to "Japanese".

Translation is slow

Switch to a faster model or provider. With Ollama, use a smaller model (qwen2.5:3b). Fast Start gets initial results on screen sooner while the rest translates in the background.

Nothing happens at all

Check Tampermonkey is enabled (icon should be active). Refresh the Netflix page. Open browser console (F12 → Console) and look for [SubTranslator] messages.

Privacy & Legal

Your data: API keys stay in your browser (Tampermonkey storage). Zero analytics or tracking.

Cloud providers: Subtitle text is sent to Gemini/Groq/Mistral/etc. for translation — similar to pasting into Google Translate. Netflix subtitle text is copyrighted material; sending it to a third-party server may be against Netflix's TOS and/or applicable law depending on your jurisdiction. Use Ollama for fully local, private translation if this is a concern.
